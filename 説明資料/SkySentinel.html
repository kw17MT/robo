i. 被写界深度

私のゲームでは、タイトルに被写界深度を適用しています。
背景にぼかしをいれることで、ロボットに注目させるとともに、アセットの作成とインスタンス生成を削減させることができました。


今回は、カメラの注視点となるロボットから一定距離離れた場所に被写界深度を掛けるようにしています。
ディファードレンダリング実行時に得た深度値マップとゲーム画面全体にブラーを掛けたテクスチャを用いて表現します。
作成過程を以下に記します。

1.ゲーム画面となるテクスチャにブラーを掛けたテクスチャを作成する。
メインのゲーム画面を取得し、X座標方向、Y座標方向にガウシアンブラーを掛けます。
以降の過程でブラーを適用したテクスチャを選択的にゲーム画面に適用していきます。




2.深度値マップから深度値を取得。
モデル描画時に計算するワールド座標系のZ値を深度値とします。その深度値をこの深度値マップに保存しています。

このテクスチャから深度値をサンプリングし、正規化デバイス座標のzとして利用します。

NormalizedDeviceCoord.z = Sample(sampler, In.uv).z;

正規化デバイス座標のxy座標について0～1の範囲で書き込まれるUV座標系から、-1～1の範囲で書き込まれる正規化デバイス座標系に変換する。

NormalizedDeviceCoord.xy = float2 (uv) * float2(2.0f, -2.0f) + float2(-1.0f, 1.0f);

3.正規化デバイス座標系をワールド座標に変換する。
定めたxyz座標にプロジェクション逆行列とビュー逆行列を乗算する。
プロジェクションビュー行列は学校内製エンジンのカメラクラスにあるため、その逆行列を利用します。

NormalizedDeviceCoord.w = 1.0f;
worldPos = float4x4(ProjectionInverseMatrix) * float4x4(ViewInverseMatrix) * NormalizedDeviceCoord;

計算結果のxyz座標をw座標で除算して透視除算を行う。
透視除算を行うことでカメラに写っていないワールド座標を除外する。

worldPos.xyz = worldPos.w;

4.求めたワールド座標からカメラ注視点の座標までの距離を計算する。
被写界深度を適用したい距離未満のピクセルには被写界深度を起こさないようにする。

作成過程を以下に記します。

2.ゴースト表現のためにブラーを掛ける。

レンズゴーストが発生する場所の条件が以下の画像です。
輝度の高い物体（太陽）から画像の中心へ向かうように発生します。

iii. スプラットマップ

このゲームの地面は、スプラットマップを利用してテクスチャを局所的に変更しています。
まず、起伏生成には1枚のノイズ画像と3dsMaxの機能のディスプレイスメントマッピングを利用しています。

使用したノイズ画像の上に起伏が高く出そうな部分、そうでない部分で赤、緑、青で色分けしたスプラットマップを用意します。

スプラットマップをシェーダー上で参照して、そのテクセルが赤色ならば砂地、緑色ならば草地、青色ならば雪原をピクセルに適用し、使用するテクスチャを切り替えています。

float4 returnColor = SplatMap.r * SandTexture + SplatMap.g * GrassTexture + SplatMap.b * SnowTexture;



iv. モーションブラー

ゲーム画面にスピード感を持たせたかったため、モーションブラーを実装しました。特にカメラを振った時や、プレイヤーと敵がすれ違う時に現れます。

モデル描画時にベロシティマップを作成します。1フレーム前のビュープロジェクション行列と現在のビュープロジェクション行列、1フレーム前のワールド座標行列をもちいて、シェーダー内で1フレーム前のスクリーン座標と現在のスクリーン座標を計算し、その差をテクスチャとして書き出します。
作成過程を以下に記します。

1.ベロシティマップの作成
i)モデル定数バッファにワールド座標の逆行列、1フレーム前のプロジェクションビュー行列、同行列の逆行列、1フレーム前のワールド座標の行列を設定する。
ii)頂点シェーダー内で現在のスクリーン座標と1フレーム前のスクリーン座標を計算する。
iii)ピクセルシェーダー内でモデルのスクリーン上の1フレーム前の座標と現在の座標を計算する。

float4 prevPosInScreen = mul(prevViewProjMatrix, float4(psIn.prevWorldPos, 1.0f));  
float4 currentPosInScreen = mul(currentViewProjMatrix,float4(psIn.worldPos.xyz, 1.0f));

iv).両方の座標の差を計算し、速度を求めベロシティマップに書き込む。

float4 Velocity = currentPosInScreen - prevPosInScreen;




実際に作成したベロシティマップの各要素が下記の通り確認できました。

2.速度とその向き応じてピクセルに色を加算する
作成したベロシティマップを参照し、速度をサンプリングします。
速度がある場合、その向きに向かってサンプリングする際のUV座標をずらして色を取得します。

float4 velocity = velocityMap.Sample(Sampler, uv);
for (int i = 0; i < SamplingNum; i++)
{        
    returnColor += gameTexture.Sample(Sampler, In.uv + (velocity.xy) * i) * weight;
}    
finalColor.w = 1.0f;

v. モノクロ化

プレイヤーのHPが低下した時、戦闘エリアの外に出ようとしている時、UI以外のゲーム画面にモノクロ化を施しています。
プレイヤーに危険が迫っている事を知らせる役割をしています。 エリアでの判定では、エリア圏内にいるかどうかでモノクロ化の制御を行っています。 HPでの判定では、プレイヤーの残りHP割合に応じて段々とモノクロ化するようにしています。 実装に関しては、UIが入っていないゲーム画面テクスチャを対象に色要素RGBにITU-R Rec BT.601規格のグレースケールを使用しています。
残りHPの割合に応じてモノクロ化させたい度合いに変化を持たせたいため各色にC++側で指定したモノクロ率を掛け合わせています。

float Color = (0.299f * R * monochromeRate) + (0.587f * G * monochromeRate)+ (0.114f * B * monochromeRate);

6. こだわった点・苦労した点

i. タイトル画面中のロボット観覧

タイトル画面は画像ではなく、ドックにロボットが待機している状態にし、コントローラー入力で角度を変えてロボットを見ることができます。自分が操作する機体をじっくりと観察することで愛着がわくのではないかと考えました。
おまけ要素ではありますが、整備員をあまり目立たないところに立たせてみました。

ii. 出撃時の演出

待機していたドックからの出撃をイメージして制作しました。
カメラの位置を変え、射出口をプレイヤーに見せるシーン。ロボットのエンジン起動（バーニア噴射）シーン。出撃シーンと遷移していくように制作しました。
エンジン起動シーンではバーニアエフェクトをわざと大きくさせて目立たせ、噴射の影響でじわじわと前進してしまう様子を表現しています。
出撃シーンではロボットを一気に射出させるとともに、太陽光も相まってすぐさま遠くに行ってしまうように見せています。

iii. 移動時のプレイヤーとカメラの位置関係

プレイヤーが右に移動すると、画面上でも右に寄らせたり、上昇したならば、上方向にずらして描画したりしています。
特にダッシュを行った直後は著しく画面端によせてスピード感がでるように工夫を施しました。
単純にプレイヤーの背後を追いかけるカメラと比べると、どちらに移動しているのか分かりやすい効果が生まれていると思います。

iv. uvスクロール

プレイヤーのHPバー、エネルギーバーに半透明な画像を走らせて装飾を施しています。
シェーダーに装飾用画像を送り、UV座標をサンプリングする際のオフセット値をC++側から送り制御できるようにしています。
バーを直置きしている時と比較すると存在感が増し、デザインも向上したのではないかと感じています。

v. 射撃時のライト

マシンガンを打つ際、マシンガンからプレイヤーに向けてオレンジ色のスポットライトを点滅させるようにしています。
銃口のノズルフラッシュの影響を受けているイメージで実装しました。

vi. 敵の移動

敵の移動を実装するにあたり、不自然さがなくかつ単調になり過ぎないように心がけて実装しました。
このゲームの参考にした作品を観察し、コードに落とし込みました。
敵の基本的な動き方としてはプレイヤーに迫ってくるように移動します。プレイヤーに十分に接近後、または近づきすぎている状況下でプレイヤーの背後をとるか、そのまま攻撃し続けるか、いったん距離を離すかの行動をとります。

vii. スプライトの色変化

プレイヤーが場外に出てしまいそうなとき、戦闘エリアに戻るように促す画像に工夫を加えました。
極力目立つように生成していますが、事の重大さを伝えるために、シェーダー側で画像の色を周期的に変化させています。こちらもC++側で色の変化の度合いを送り、色を変更させるような作りにしています。
